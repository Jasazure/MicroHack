# MicroHack Challenges – Agent vs. Human Intervention Analysis

This document summarises every challenge set found in this repository and classifies each challenge step as either **automatable by an AI/scripting agent** or **requiring human intervention** (portal UI interaction, physical access, subscription approval, or decision-making).

---

## Legend

| Symbol | Meaning |
|--------|---------|
| 🤖 | Can be fully automated (CLI, Bicep, Terraform, scripts) |
| 🧑 | Requires human intervention (portal UI, physical install, decisions, subscriptions) |
| ⚡ | Partial automation – setup can be scripted but validation/review requires a human |

## Key Human Intervention Points (Never Delegate)

The following actions across all MicroHacks **must always be performed by a human**:

1. **Azure Marketplace / Oracle Cloud subscription approvals** – Require accepting legal terms and billing agreements.
2. **Azure Migrate Appliance deployment** – Requires downloading, installing, and registering software inside a VM.
3. **DR failover and failback decisions** – Critical production actions that must be human-approved.
4. **Business Case and migration strategy decisions** – Require organisational context and sign-off.
5. **Workbook and dashboard customisation** – Portal-only visual canvas tools.
6. **Attestation token review** – Requires human inspection of cryptographic output.
7. **ArcBox/LocalBox pre-existing infrastructure** – Requires provisioned lab environment before the challenge can begin.

---

## Key Automation Opportunities (Safe to Delegate to Agent)

The following actions are safe to automate with scripts or an AI agent:

1. **Bicep / Terraform / ARM deployments** – All landing zone preparation steps.
2. **Azure Policy and RBAC assignments** – Via `az policy` and `az role` CLI commands.
3. **Key Vault, CMK, and TLS enforcement** – Via `az keyvault` and `az storage` CLI commands.
4. **Defender for Cloud enablement** – Via `az security pricing` CLI commands.
5. **AKS cluster creation and app deployment** – Via `az aks` and `kubectl` commands.
6. **Container builds and ACR pushes** – Via `docker build` and `az acr` commands.
7. **Log Analytics workspace, DCRs, and alert rules** – Via `az monitor` CLI commands.
8. **Helm chart deployments** – Via `helm install` commands.
9. **App Service code deployments** – Via `az webapp deploy` commands.
